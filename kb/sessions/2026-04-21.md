## Session Summary
- Studied the prompt-path boundary around `AgentSession.prompt()` and clarified what is app-layer preflight versus generic agent runtime behavior.
- Key insights gained:
  - `AgentSession.prompt()` does much more than forwarding text: extension command dispatch, input interception, skill/template expansion, queue routing during streaming, pending bash flush, model/auth checks, compaction checks, message assembly, extension `before_agent_start` hooks, then delegation to `agent.prompt(messages)`.
  - A single prompt/run is not just one LLM call. It can span multiple turns when tool calls, steering, or follow-up messages keep the loop alive.
  - Slash handling order is intentional: extension commands get first shot; `/skill:name` is only expanded as a skill if no extension command named `skill:name` exists.
  - `preflightResult(success)` is an internal acceptance/rejection callback for callers such as RPC mode, not a full prompt-success callback.
  - Pending bash results are buffered during streaming and flushed later to preserve transcript ordering around assistant/tool messages.
  - `waitForRetry()` makes `AgentSession.prompt()` settle only after any session-level auto-retry finishes.

### What We Studied

- Pi run modes at a conceptual level: interactive, print, JSON, RPC, SDK
- Session persistence basics and when `--no-session` matters

### Key Insights Gained
- `packages/coding-agent/README.md` is the real first deep orientation doc for learning Pi as an application
- JSON mode emits Pi-normalized runtime/session events rather than raw provider protocol payloads
- RPC mode is best understood as Pi running as a controllable child process over stdin/stdout JSONL
- RPC is the right integration path for non-Node hosts like Java; SDK is the in-process path for Node/TypeScript
- A PID is not an RPC address; if another program needs to control Pi reliably, it should normally spawn the Pi RPC process itself

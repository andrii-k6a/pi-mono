## Key Insights
- Skills are prompt-layer resources, not provider-native APIs or tool-call contracts
- Automatic skill use works by advertising skill metadata in the system prompt and letting the model `read` the full skill file on demand
- Explicit `/skill:name` expands into a single user message containing the full skill block before any trailing arguments
- `AGENTS.md` loading is implemented in `resource-loader.ts` by loading global context first, then walking upward from `cwd` to filesystem root and ordering files outermost-to-innermost
- Pi's tmux philosophy is about refusing a built-in background-job subsystem, not forbidding the agent from invoking tmux commands
